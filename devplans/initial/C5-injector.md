# C5. Injector 개발계획서

| 항목 | 내용 |
|------|------|
| 구성요소 ID | C5 |

| 계층 | Core Engine (Windows 전용) |

| 상위 문서 | `README.md` |

---

## 1. 목적/책임
가공된 텍스트를 **현재 포커스된 입력창(커서 위치)** 에 자동 입력한다. 한글 안정성이 최우선.

- SendInput 유니코드 직접 주입.
- 클립보드 백업→붙여넣기→복원 폴백.
- 상황(대량/실시간)에 따른 전략 자동 선택.
- 실시간 replacement(직전 주입 텍스트 대체) 지원.

**책임이 아닌 것**: 언제 주입할지 결정(C4/C10), 텍스트 내용 가공(C17).

---

## 2. 선택 근거와 대안 검토
- **SendInput + KEYEVENTF_UNICODE 우선**: IME를 우회해 유니코드 코드포인트를 직접 주입 → 한글 조합 깨짐이 원천적으로 없고 클립보드를 훼손하지 않음. 실시간 청크에 적합.
- **클립보드 + Ctrl+V 폴백**: 대량 텍스트에 빠르고 안정적. 단, 반드시 기존 클립보드를 백업/복원.
- **pynput.type() 단일 방식 배제**: 한글 IME 조합에서 깨짐 발생 → 부적합(대안에서 제외).
- **자동 선택**: 길이 임계값/모드에 따라 unicode ↔ clipboard 전환.

---

## 3. 공개 인터페이스 계약
- `inject(text, method=auto|unicode|clipboard, options) -> InjectResult`.
- `replace_last(new_text, prev_text_len) -> InjectResult`: 실시간 교체(직전 주입분 삭제 후 새로 주입).
- `capabilities() -> InjectCapabilities`.

`InjectResult` = { success, method_used, error? }.
`options` = { press_enter?: bool(자동 전송), per_char_delay?: int, target_check?: bool }.

---

## 4. 입력/출력 데이터와 상태 전이
- 입력: 텍스트 + 방식/옵션.
- 출력: 활성 창의 커서 위치에 텍스트 입력 + 결과 상태.
- 상태: 무상태(stateless) 동작이 기본. 단, replacement를 위해 "직전 주입 길이"를 호출자(C4/C10)가 관리하여 전달.

---

## 5. 의존 구성요소와 호출 관계
- **호출자**: C4 Pipeline(주입 요청), C10 SessionManager(주입 실행 조율).
- **의존**: Win32 API(ctypes/pywin32), C20 Logging.
- **협조**: C9 HotkeyManager(자동 전송 Enter 옵션), C11 Config(기본 방식/딜레이).

---

## 6. 내부 설계
### 6.1 구조
- `injector.py`: 전략 선택 + 공개 API.
- `unicode_input.py`: SendInput 유니코드 시퀀스 구성/전송(서로게이트 페어 처리).
- `clipboard_input.py`: 클립보드 백업/설정/붙여넣기/복원.
- `foreground.py`: 활성 창/포커스 확인, 관리자 권한 대상 판별(가능 범위).

### 6.2 유니코드 주입 절차
1. 텍스트를 코드포인트 시퀀스로 변환(BMP 밖 문자는 서로게이트 페어).
2. 각 문자에 대해 keydown/keyup 유니코드 이벤트를 SendInput 배치로 전송.
3. 과도한 이벤트로 인한 유실 방지를 위해 청크 단위 전송 + 미세 지연(옵션).
4. `press_enter` 옵션 시 마지막에 Enter 전송.

### 6.3 클립보드 주입 절차
1. 현재 클립보드 내용 백업(가능한 포맷 보존).
2. 주입 텍스트를 클립보드에 설정.
3. Ctrl+V 전송.
4. 짧은 지연 후 원래 클립보드 복원.
5. 복원 실패도 로그로 남김.

### 6.4 전략 자동 선택 규칙(기본안)
- 실시간 세그먼트/짧은 텍스트 → unicode.
- 길이 임계값(예: 수백 자) 초과 → clipboard.
- 사용자가 방식 강제 지정 시 우선.

### 6.5 replacement(실시간 교체)
- 직전 주입 길이만큼 Backspace 전송 후 새 텍스트 주입.
- 사용자가 그 사이 다른 곳을 클릭했으면 안전 판단 불가 → 옵션으로 비활성/보수적 동작.

---

## 7. 오류/폴백/재시도 정책
- unicode 주입 실패/유실 감지 시 clipboard로 폴백(옵션).
- 활성 창 없음/입력 불가(관리자 권한 창 등) → 실패 반환 + 명확한 안내(권한 제약 문서화).
- 클립보드 잠금(다른 앱 점유) → 재시도 후 실패 시 안내.
- 대량 텍스트 유니코드 유실 위험 → clipboard 우선.

---

## 8. 성능 목표
- 짧은 문장 주입 체감 지연 최소(수십 ms).
- 대량 텍스트는 clipboard로 즉시 붙여넣기.
- per-char 지연은 필요한 앱에서만 최소 적용.

---

## 9. 보안/프라이버시 고려사항
- 클립보드에 잠깐 올라간 민감 텍스트는 복원으로 즉시 제거.
- 주입 텍스트 원문 로그 미기록.
- 자동 전송(Enter)은 오작동 시 위험하므로 기본 비활성 + 명시적 설정.

---

## 10. 테스트 항목
- 단위: 서로게이트 페어 변환, 이벤트 시퀀스 구성, 클립보드 백업/복원 로직(mock).
- 통합(수동 필수): 메모장/브라우저/IDE/메신저에 한글·특수문자·이모지 주입 손실 0% 검증.
- 엣지: 활성 창 없음, 관리자 창, 초대량 텍스트, 빠른 연속 주입, 교체 중 포커스 이동.

---

## 11. 관련 Phase와 구현 범위
- **P0**: 유니코드 한글 주입 손실 0% 검증(프로젝트 최상위 리스크).
- **P1**: 배치 결과 주입(unicode+clipboard 자동 선택).
- **P2**: 실시간 청크 주입 + replacement.

---

## 12. 미결정 사항
- 전략 전환 길이 임계값.
- replacement 안전 조건(포커스 이동 감지 방법).
- 특정 앱 예외 목록 관리 방식.
