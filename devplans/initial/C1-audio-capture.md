# C1. AudioCapture 개발계획서

| 항목 | 내용 |
|------|------|
| 구성요소 ID | C1 |
| 이름 | AudioCapture (오디오 캡처) |
| 계층 | Core Engine |
| 관련 Phase | P0(검증), P1(배치), P2(스트림/VAD), P5(튜닝) |
| 상위 문서 | `README.md` (마스터 계획) |

---

## 1. 목적/책임
마이크 입력을 캡처하여 STT가 소비할 수 있는 표준 오디오 데이터로 변환·공급한다.

- 마이크 장치 열거/선택/열기/닫기.
- 원시 오디오를 **16kHz, mono, 16-bit PCM** 표준 내부 포맷으로 변환.
- 두 가지 공급 방식 제공: (a) 단위 녹음(Batch) 버퍼, (b) 실시간(Streaming) 청크.
- 실시간 모드에서 VAD(음성 활동 감지)로 발화 세그먼트 경계를 판정.
- 입력 레벨(파형/피크) 계산값을 UI에 제공.

**책임이 아닌 것**: STT 변환(C2), 녹음 시작/종료 트리거 결정(C9 HotkeyManager), 파일 저장(C6/C10).

---

## 2. 선택 근거와 대안 검토
- **표준 포맷을 16kHz mono 16-bit로 고정**: Whisper 계열 STT의 기대 입력이며, 모든 Provider가 공통으로 소비 가능. 변환 지점을 한 곳(C1)에 모아 중복 방지.
- **오디오 라이브러리**: `sounddevice`(PortAudio 기반)를 1순위 후보로 한다. 콜백 기반 스트리밍이 간결하고 패키징이 비교적 안정적이다. 대안 `pyaudio`는 설치·휠 이슈가 잦아 후순위. **P0에서 두 라이브러리의 장치 호환성·패키징을 실측 후 확정**한다.
- **VAD**: Silero VAD(1순위). 정확도가 높고 한국어 발화에도 무난. 대안 WebRTC VAD는 가볍지만 경계 정확도가 낮아 보조/폴백용.
- **리샘플링**: 장치 기본 샘플레이트(예: 44.1k/48k)에서 16k로 다운샘플. `soxr` 또는 라이브러리 내장 리샘플러 사용. P0에서 품질/성능 비교.

---

## 3. 공개 인터페이스 계약
다른 구성요소가 호출하는 개념적 API(언어 시그니처가 아닌 계약).

- `list_devices() -> [DeviceInfo]`: 사용 가능한 입력 장치 목록(id, 이름, 기본 여부, 지원 샘플레이트).
- `select_device(device_id)`: 사용할 장치 지정. 미지정 시 시스템 기본 장치.
- `start_batch() -> RecordingHandle`: 단위 녹음 시작. 내부 버퍼에 누적.
- `stop_batch(handle) -> AudioBuffer`: 녹음 종료, 표준 포맷 오디오 버퍼 반환.
- `start_stream(on_segment, on_level) -> StreamHandle`: 실시간 시작. 콜백으로 세그먼트/레벨 전달.
- `stop_stream(handle)`: 실시간 종료.
- `cancel(handle)`: 녹음 취소(버퍼 폐기, 산출물 없음).

**이벤트/콜백**
- `on_level(peak, rms)`: UI 파형 표시용(주기 예: 30~60ms).
- `on_segment(AudioBuffer, is_final)`: VAD가 감지한 발화 세그먼트.
- `on_error(AudioError)`: 장치 분리, 접근 거부 등.

---

## 4. 입력/출력 데이터와 상태 전이
### 데이터
- 입력: OS 마이크 스트림(장치 기본 포맷).
- 출력: `AudioBuffer` = { pcm_bytes, sample_rate=16000, channels=1, sample_width=2, duration_ms }.

### 상태 전이
```
idle → (start_batch) → recording → (stop_batch) → idle
idle → (start_stream) → streaming → (stop_stream) → idle
recording/streaming → (cancel) → idle
recording/streaming → (device error) → error → (복구/알림) → idle
```
- 동시에 하나의 활성 캡처만 허용(batch와 stream 동시 불가). 중복 요청은 거부하고 오류 반환.

---

## 5. 의존 구성요소와 호출 관계
- **호출당하는 곳**: C10 SessionManager(녹음 시작/종료), C12 UI(레벨 표시).
- **직접 의존**: 없음(순수 Core). VAD 모델 파일은 C18 ModelManager 또는 번들 리소스에서 로드.
- **하위 전달**: 산출된 `AudioBuffer`는 C4 Pipeline → C2 STT로 전달.
- **로깅**: C20에 장치 열기 실패/리샘플 오류 기록(민감정보 없음).

---

## 6. 내부 설계
### 6.1 구조(모듈 분리)
- `device.py`: 장치 열거/선택/열기.
- `capture_batch.py`: 콜백에서 링버퍼에 누적, stop 시 병합.
- `capture_stream.py`: 콜백에서 프레임을 VAD 프로세서로 전달.
- `vad.py`: Silero VAD 래퍼(프레임 → 발화/침묵 판정, 세그먼트 조립).
- `resample.py`: 임의 샘플레이트 → 16k mono 변환.
- `format.py`: `AudioBuffer` 정의 및 변환 유틸.

### 6.2 처리 절차 (실시간)
1. 콜백이 장치 프레임 수신(장치 포맷).
2. 모노 다운믹스 + 16k 리샘플.
3. 20~30ms 프레임 단위로 VAD 판정.
4. "침묵→발화" 전이에서 세그먼트 시작, "발화→침묵(N ms 지속)"에서 세그먼트 종료.
5. 종료된 세그먼트를 `on_segment(..., is_final=True)`로 방출.
6. 레벨 값은 별도 주기로 `on_level`.

### 6.3 파라미터(설정 노출, C11)
- VAD 민감도(threshold), 최소 발화 길이, 종료 침묵 길이(hangover), 최대 세그먼트 길이(강제 컷).
- 입력 장치 id, 입력 게인(선택).

## 7. 오류/폴백/재시도 정책
- **장치 접근 거부(Windows 마이크 권한)**: 명확한 사용자 안내 메시지 + 설정 안내. 자동 재시도 금지.
- **장치 분리(런타임)**: 진행 중 세그먼트는 확보분까지 방출, `error` 상태로 전이, UI 알림.
- **리샘플 실패**: 해당 청크 드롭 후 로그, 반복되면 스트림 중단.
- **VAD 모델 로드 실패**: WebRTC VAD로 폴백하거나, 폴백 불가 시 실시간 모드 비활성 + 배치 모드만 허용.

---

## 8. 성능 목표
- 콜백 처리 지연: 프레임당 처리 5ms 이내(리샘플+VAD 포함).
- 실시간 세그먼트 방출 지연: 발화 종료 침묵 감지 후 100ms 이내.
- 유휴 시(캡처 미동작) CPU 사용 0%에 근접.
- 메모리: 최대 세그먼트 길이 * 안전계수 이내 버퍼.

---

## 9. 보안/프라이버시 고려사항
- 오디오 원음은 메모리에 두는 것을 기본으로 하고, 파일로 남길지는 C10/C11 정책을 따른다.
- 로그에 오디오 데이터/경로 원문을 남기지 않는다(장치명 정도만).

---

## 10. 테스트 항목
- 단위: 리샘플 정확도(길이/샘플레이트), 포맷 변환, VAD 세그먼트 경계(합성 오디오로 침묵/발화 케이스).
- 통합: 실제 마이크로 배치 녹음 → 버퍼 길이/포맷 검증.
- 엣지: 장치 없음, 권한 거부, 녹음 중 장치 분리, 초장시간 발화(강제 컷).
- 성능: 콜백 지연/세그먼트 지연 실측(NFR 대조).

---

## 11. 관련 Phase와 구현 범위
- **P0**: `sounddevice` vs `pyaudio` 실측, 배치 녹음 + 16k 변환 검증, 실시간 지연 측정.
- **P1**: 배치 녹음 안정화(단위 녹음 모드).
- **P2**: 실시간 스트리밍 + Silero VAD 세그먼트.
- **P5**: 노이즈 상황 튜닝, 성능 최적화.

---

## 12. 미결정 사항
- 오디오 라이브러리 최종 선택(P0 결과).
- 리샘플러 선택(soxr vs 내장).
- 노이즈 게이트/전처리(DSP)를 C1에 둘지 별도 백로그로 둘지.
- 입력 게인 수동 조절 노출 여부.
