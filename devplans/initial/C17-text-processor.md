---

## 1. 목적/책임
STT 결과(1차)를 주입/저장 전에 **결정적 규칙**으로 다듬는다. LLM이 아닌, 예측 가능한 후처리 담당.

- 한국어 정규화(공백/문장부호/숫자 표기 등).
- 사용자 사전 치환(고유명사/오인식 교정).
- 스니펫 확장(단축어 → 전체 문구).
- stage/컨텍스트별 전처리(예: 주입 대상 앱별 규칙 훅).

**책임이 아닌 것**: LLM 교정(C3), STT(C2), 주입(C5).

---

## 2. 선택 근거와 대안 검토
- **독립 구성요소 필요**: 한국어 품질/업무 편의는 STT도 LLM도 아닌 중간 후처리 책임. 분리해야 테스트/온오프가 명확.
- **결정적 규칙 우선**: LLM과 중복되지 않도록 규칙 기반(치환/정규식/사전)으로 제한. 과한 자동수정은 사용자 혼란 → 토글 제공.
- 대안(LLM에 전부 위임): 지연·비용·비결정성 증가로 부적합.

---

## 3. 공개 인터페이스 계약
- `process(text, context: ProcCtx) -> ProcResult`.
- `expand_snippets(text) -> text`.
- `apply_dictionary(text) -> text`.
- `normalize_ko(text) -> text`.

`ProcCtx` = { stage, source, target_app?, options{normalize, dictionary, snippets} }.
`ProcResult` = { text, applied[규칙 목록] }.

---

## 4. 입력/출력 데이터와 상태 전이
- 입력: 원시/STT 텍스트 + 컨텍스트.
- 출력: 후처리 텍스트 + 적용된 규칙 목록(디버그/설명용).
- 무상태(사전/설정은 C6/C11에서 로드).

---

## 5. 의존 구성요소와 호출 관계
- **호출자**: C4 Pipeline(stage1 직후), 필요 시 재가공 경로.
- **의존**: C6 Store(DictionaryRepo), C11 Config(옵션), C20 Logging.

---

## 6. 내부 설계
### 6.1 구조
- `processor.py`: 파이프 순서 조립(정규화→사전→스니펫).
- `normalize_ko.py`: 한국어 규칙(연속 공백, 문장부호 간격, 숫자/단위, 문장 끝 처리 등).
- `dictionary.py`: 사전 치환(단어 경계/대소문자/우선순위).
- `snippets.py`: 단축어 확장("슬래시 시그" 같은 STT 변형 정규화 포함).
- `rules.py`: 규칙 정의/우선순위.

### 6.2 처리 순서(기본)
1. 기초 정규화(트림, 중복 공백, 제어문자 제거).
2. 사전 치환(오인식 고유명사 교정).
3. 스니펫 확장(사용자 단축어).
4. 한국어 표기 정규화(문장부호/띄어쓰기 보정 — 보수적으로).
5. 결과 + 적용 규칙 반환.

### 6.3 안전 원칙
- 각 단계는 개별 on/off 가능(설정).
- 과교정 방지: 의미 변경 가능성이 있는 규칙은 기본 off.
- 규칙은 우선순위/충돌 해결이 명확해야 함(사전 > 스니펫 순서 등).

---

## 7. 오류/폴백/재시도 정책
- 규칙 적용 중 예외 발생 시 해당 규칙만 건너뛰고 원문 유지(전체 실패 금지).
- 사전 로드 실패 시 사전 단계만 비활성 + 로그.

---

## 8. 성능 목표
- 실시간 경로에 들어가므로 세그먼트당 수 ms 이내.
- 사전/스니펫은 인덱싱/캐시로 빠른 매칭.

---

## 9. 보안/프라이버시 고려사항
- 사전/스니펫에 민감 정보가 있을 수 있음 → 진단 export 포함 여부 고지.
- 처리 텍스트 로그 기본 미기록.

---

## 10. 테스트 항목
- 단위: 각 규칙 단위 입력/출력, 우선순위/충돌, 토글 동작.
- 통합: Pipeline stage1 직후 적용 결과.
- 엣지: 빈 문자열, 사전 대량, 겹치는 스니펫, 특수문자.

---

## 11. 관련 Phase와 구현 범위
- **P2**: 정규화/사전/스니펫 기본 구현 + Pipeline 연동.
- **P3**: 규칙 확장, 앱별 컨텍스트 훅.

---

## 12. 미결정 사항
- 한국어 정규화 규칙의 범위(어디까지 보수적으로).
- 스니펫 트리거 문법.
- 앱별(target_app) 규칙 도입 여부.
