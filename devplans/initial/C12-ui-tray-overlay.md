---

## 1. 목적/책임
상시 실행되는 트레이 앱과 녹음 상태를 보여주는 작은 오버레이 바를 제공한다. Win+H 스타일의 심플 UX.

- 시스템 트레이 아이콘/메뉴.
- 오버레이 바: 상태(대기/녹음/처리/오류) + 파형 + 빠른 모드 전환.
- 앱 진입점(설정/작업대/온보딩 열기, 종료).

**책임이 아닌 것**: 처리 로직(Application/Core), 설정 편집 화면(C14), 작업대(C13).

---

## 2. 선택 근거와 대안 검토
- **트레이 + 소형 오버레이**: "기본 GUI는 심플하게" 요구 + 받아쓰기 UX 벤치마크.
- **UI는 이벤트 구독만**: UI가 처리 로직을 갖지 않고 Application 이벤트만 반영(테스트/재사용).
- 대안(항상 큰 창): 심플 요구에 반해 배제.

---

## 3. 공개 인터페이스 계약
- 구독: `on_session_state`, `on_level`, `on_pipeline_error`, `on_inject`(피드백).
- 발신(사용자 액션): `request_record_toggle`, `request_cancel`, `change_mode(mode_id)`, `open_settings/workbench/onboarding`, `quit`.

---

## 4. 입력/출력 데이터와 상태 전이
- 입력: Application 상태/레벨 이벤트.
- 출력: 화면 표시 + 사용자 액션 이벤트.
- 오버레이 상태: hidden ↔ idle ↔ recording(파형) ↔ processing(스피너) ↔ error(짧은 알림).

---

## 5. 의존 구성요소와 호출 관계
- **의존**: C10 SessionManager(상태/제어), C1 레벨(파형), C7 Mode(빠른 전환), C11 Config.
- **엶**: C13 Workbench, C14 Settings, C21 Onboarding.

---

## 6. 내부 설계
### 6.1 구조
- `tray.py`: 트레이 아이콘/메뉴/알림.
- `overlay.py`: 프레임리스 상시 오버레이(위치/투명도/항상 위).
- `waveform.py`: 레벨 → 파형 렌더.
- `state_view.py`: 상태별 표시 전환.

### 6.2 동작 원칙
- 모든 무거운 작업은 워커(신호로 결과 수신). UI 스레드 블로킹 금지.
- 오버레이는 드래그 이동/자동 숨김 옵션.
- DPI/다중 모니터 대응(위치 보정).

### 6.3 상태 표시 매핑
- recording: 파형 + 경과시간.
- processing: 진행 표시(단계 텍스트 옵션).
- error: 트레이 알림 + 오버레이 짧은 경고.

---

## 7. 오류/폴백/재시도 정책
- 이벤트 유실/지연 시 마지막 상태 유지, 비정상 표시 방지.
- 오버레이 렌더 실패 시 트레이 알림으로 폴백.

---

## 8. 성능 목표
- 파형 갱신 30~60fps 이내로 제한(CPU 절약).
- 유휴 시 렌더 정지.

---

## 9. 보안/프라이버시 고려사항
- 화면에 원문 텍스트 노출 최소(오버레이엔 상태 위주).

---

## 10. 테스트 항목
- 단위: 상태→표시 매핑, 액션 이벤트 발신.
- 통합: 실제 세션 상태 반영, 모드 전환.
- 엣지: 다중 모니터/DPI, 빠른 상태 전환, 장시간 실행.

---

## 11. 관련 Phase와 구현 범위
- **P1**: 트레이 + 기본 오버레이(상태/파형) + 진입 메뉴.
- **P2**: 빠른 모드 전환, 처리 단계 표시.

---

## 12. 미결정 사항
- 오버레이 기본 위치/자동 숨김 동작.
- 파형 시각화 스타일.
- 트레이 알림 사용 범위.
