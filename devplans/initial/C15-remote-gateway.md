# C15. RemoteGateway 개발계획서

| 항목 | 내용 |
|------|------|
| 구성요소 ID | C15 |
| 이름 | RemoteGateway (원격 녹음 위성) |
| 계층 | 통합 (FastAPI + PWA + Tunnel) |
| 관련 Phase | P4(핵심) |
| 상위 문서 | `README.md` |

---

## 1. 목적/책임
모바일/웹을 **녹음 전용 위성**으로 연결한다. 폰 브라우저에서 녹음한 오디오를 Windows 허브로 전달해 파이프라인에 투입한다.

- 로컬 FastAPI 서버(오디오 수신).
- 초간단 녹음 PWA(폰 브라우저).
- QR/PIN 페어링.
- 터널(Cloudflare)로 HTTPS/외부 접속.

**책임이 아닌 것**: 모바일에서 타 앱 주입(범위 제외), STT/LLM(C2/C3), 세션 로직(C10 재사용).

---

## 2. 선택 근거와 대안 검토
- **녹음 전용으로 축소**: 사용자가 모바일/웹 입력 기능을 요구하지 않음(녹음만). 복잡도 대폭 감소.
- **FastAPI + PWA + Cloudflare Tunnel**: 모바일 마이크는 HTTPS 필수 → 터널이 자동 HTTPS + QR 접속 해결(airmic 벤치마크). 서버 비용 0.
- 대안(중앙 중계 서버): 상용화 시 검토(README 미결정).
- 대안(로컬 IP 직결): HTTPS/네트워크 불일치 문제로 부적합.

---

## 3. 공개 인터페이스 계약
- 서버 제어: `start_server()`, `stop_server()`, `get_access_info() -> {url, qr, pin}`.
- HTTP 엔드포인트(위성용): 페어링, 오디오 업로드(파일/청크), 상태.
- 허브 연동: 수신 오디오 → `C10.submit_remote(audio, mode)`.

---

## 4. 입력/출력 데이터와 상태 전이
- 입력: 폰 브라우저 오디오(WebM/WAV 등) + 페어링 정보.
- 출력: 표준 오디오로 변환 후 세션 생성.
- 서버 상태: stopped → starting(tunnel) → running(paired?) → stopped.

---

## 5. 의존 구성요소와 호출 관계
- **의존**: C10 SessionManager(원격 세션), C1(포맷 변환 재사용), C19(페어링 시크릿), C11(설정), C20(로깅), Cloudflare Tunnel.
- **호출자**: C12/C14 UI(서버 켜기/접속 정보 표시).

---

## 6. 내부 설계
### 6.1 구조
- `server.py`: FastAPI 앱/엔드포인트.
- `pairing.py`: PIN/QR 발급·검증, 세션 토큰.
- `tunnel.py`: Cloudflare Tunnel 실행/URL 획득.
- `pwa/`: 녹음 웹앱(정적 파일: 녹음 버튼/업로드).
- `ingest.py`: 업로드 오디오 → 표준 포맷 변환 → C10 제출.

### 6.2 페어링 절차
1. 서버 시작 → 터널 URL + PIN 발급 → UI에 QR 표시.
2. 폰이 QR 접속 → PIN 입력 → 세션 토큰 발급.
3. 이후 업로드는 토큰 인증.

### 6.3 오디오 수신 절차
1. PWA에서 녹음(getUserMedia) → blob 업로드(또는 청크).
2. 서버가 토큰 검증 → 포맷 변환(C1) → `C10.submit_remote`.
3. 허브 파이프라인이 처리(주입은 PC 기준).

### 6.4 보안 기본
- 인증 없는 업로드 차단(토큰 필수).
- 업로드 크기/시간 제한.
- 터널 URL은 세션 한정, 종료 시 폐기.

---

## 7. 오류/폴백/재시도 정책
- 터널 실패: 재시도 + 로컬 네트워크 폴백 안내(선택).
- 네트워크 불안정: 청크 재전송/타임아웃.
- 페어링 실패/만료: 재페어링 유도.

---

## 8. 성능 목표
- 파일 단위 업로드는 안정 우선. 실시간 스트리밍은 후순위(터널 지연 고려).
- 업로드→세션 생성 지연 최소.

---

## 9. 보안/프라이버시 고려사항
- 오디오가 터널을 통해 전송됨을 사용자에게 고지.
- 토큰/PIN은 C19 관리, 로그 미기록.
- 서버는 필요할 때만 실행(상시 오픈 금지).

---

## 10. 테스트 항목
- 단위: 페어링 검증, 업로드 인증, 포맷 변환.
- 통합: 폰 브라우저 녹음 → 허브 artifact 생성 E2E.
- 엣지: 미인증 업로드, 대용량, 터널 끊김, 동시 다중 위성.

---

## 11. 관련 Phase와 구현 범위
- **P4**: 전체 구현(P3까지 허브 안정화 후 착수). 그 이전엔 미구현.

---

## 12. 미결정 사항
- 파일 단위만 vs 실시간 스트리밍 지원.
- 터널 제공자 최종(Cloudflare vs 대안).
- 다중 위성 동시 연결 허용 여부.
