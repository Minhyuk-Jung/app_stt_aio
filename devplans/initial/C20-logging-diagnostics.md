# C20. Logging/Diagnostics 개발계획서

| 항목 | 내용 |
|------|------|
| 구성요소 ID | C20 |
| 이름 | Logging/Diagnostics (로깅·진단) |
| 계층 | Application (교차 관심사) |
| 관련 Phase | P0(얇게 시작), P3(핵심) |
| 상위 문서 | `README.md` |

---

## 1. 목적/책임
구조화 로깅과 진단 정보를 제공하여 사용자 환경 의존 문제(오디오/주입/모델)를 재현·해결 가능하게 한다.

- 구조화 로그(레벨/모듈/컨텍스트).
- 오류 이벤트 수집.
- 진단 패키지(zip) export(로그+환경정보, 민감정보 제외).
- 텔레메트리(기본 OFF, 옵트인).

**책임이 아닌 것**: 비밀정보 저장(C19, 마스킹 규칙은 사용), UI 표시(C13/C14).

---

## 2. 선택 근거와 대안 검토
- **진단 export 필요**: STT/주입/모델 문제는 환경 의존이 커서(벤치마크 KusPus 사례) 사용자 로그 수집이 중요.
- **구조화 로깅**: 문자열 로그보다 필터/분석 용이.
- **텔레메트리 옵트인**: 프라이버시 우선 제품 성격상 기본 OFF.

---

## 3. 공개 인터페이스 계약
- `get_logger(module) -> Logger`(debug/info/warn/error, 컨텍스트 필드).
- `report_error(error, context)`.
- `export_diagnostics(dest_path) -> path`.
- `set_level(level)`.

---

## 4. 입력/출력 데이터와 상태 전이
- 입력: 각 구성요소 로그 호출, 오류 이벤트.
- 출력: 로그 파일(`%APPDATA%\STT-AIO\logs`), 진단 zip.
- 로그 로테이션(크기/일자) 상태 관리.

---

## 5. 의존 구성요소와 호출 관계
- **호출자**: 모든 구성요소.
- **의존**: C19(마스킹 규칙), C11(로그 레벨/경로), 파일시스템.

---

## 6. 내부 설계
### 6.1 구조
- `logging_setup.py`: 핸들러/포맷터/로테이션 구성.
- `context.py`: 세션/구성요소 컨텍스트 필드 주입.
- `diagnostics.py`: 환경정보 수집 + zip 패키징.
- `masking_hook.py`: 출력 전 C19.mask 적용.

### 6.2 로그 정책
- 레벨: 기본 INFO, 문제 시 DEBUG로 상향(설정).
- 로테이션: 크기/보관 개수 제한.
- 포맷: 시간/레벨/모듈/세션ID/메시지(+구조 필드).

### 6.3 진단 패키지 내용
- 최근 로그, 앱/OS 버전, 오디오 장치 목록, 활성 provider/모델(키 제외), 설정 스냅샷(민감 제외), GPU/CPU 정보.
- **민감정보(키/원문 텍스트/오디오)는 제외**(마스킹/필터).

---

## 7. 오류/폴백/재시도 정책
- 로그 파일 쓰기 실패: 콘솔/메모리 폴백, 앱 동작 방해 금지.
- 진단 export 실패: 부분 수집이라도 생성 + 실패 항목 표기.

---

## 8. 성능 목표
- 로깅 오버헤드 최소(비동기 핸들러 고려).
- 실시간 경로에서 과도한 DEBUG 로그 회피.

---

## 9. 보안/프라이버시 고려사항
- 모든 출력은 마스킹 훅 경유(키/토큰/원문 보호).
- 진단 export 전 "포함 내용" 사용자 고지 + 동의.
- 텔레메트리 기본 OFF, 전송 항목 명시적.

---

## 10. 테스트 항목
- 단위: 마스킹 적용, 로테이션, 컨텍스트 필드, 레벨 필터.
- 통합: 진단 zip 생성 내용 검증(민감정보 부재 확인).
- 엣지: 로그 경로 권한 없음, 대량 로그, 동시 기록.

---

## 11. 관련 Phase와 구현 범위
- **P0**: 최소 파일 로깅(오디오/모델/주입 문제 추적용).
- **P3**: 구조화/로테이션/진단 export/마스킹 완성.
- **P5**: 텔레메트리(옵트인) 검토.

---

## 12. 미결정 사항
- 로그 보관 크기/기간 기본값.
- 텔레메트리 도입 여부/항목.
- 진단 export에 프롬프트/사전 포함 여부(기본 제외 권장).
