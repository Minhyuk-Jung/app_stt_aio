| 상위 문서 | `README.md` |

---

## 1. 목적/책임
세션의 산출물(1/2/3차)을 사용자가 선택한 포맷(txt/md/docx)으로 파일 출력한다.

- 단일 artifact 또는 여러 artifact(세션 전체) 내보내기.
- 포맷별 렌더링(특히 docx 보고서/회의록 양식).
- 파일명 규칙 및 중복 처리.

**책임이 아닌 것**: 텍스트 생성(C3/C4), 저장 조회 원본(C6에서 읽음), 파일 대화상자 UI(C13).

---

## 2. 선택 근거와 대안 검토
- **txt/md/docx 3종**: 요구사항 명시. txt/md는 문자열 템플릿으로 충분, docx는 `python-docx`로 스타일 있는 문서 생성.
- **템플릿 기반**: 회의록/보고서 양식을 코드가 아닌 템플릿 규칙으로 관리해 확장 용이.
- 대안 pandoc: 강력하지만 외부 바이너리 의존이라 배포 부담 → 후순위.

---

## 3. 공개 인터페이스 계약
- `export(target: ExportTarget, format, dest_path, template?) -> ExportResult`.
- `available_templates(format) -> [TemplateInfo]`.

`ExportTarget` = { session_id, stages[선택: 1/2/3], include_meta?: bool }.
`ExportResult` = { path, format, success, error? }.

---

## 4. 입력/출력 데이터와 상태 전이
- 입력: 세션/artifact 데이터(C6) + 포맷 + 경로 + 템플릿.
- 출력: 파일(txt/md/docx).
- 무상태 동작. 실패 시 부분 파일 미생성 보장(임시 파일 후 원자적 이동).

---

## 5. 의존 구성요소와 호출 관계
- **호출자**: C13 Workbench(내보내기 액션).
- **의존**: C6 Store(내용 조회), python-docx, C20 Logging.
- **협조**: C11 Config(기본 출력 폴더/파일명 규칙).

---

## 6. 내부 설계
### 6.1 구조
- `exporter.py`: 진입점, 포맷 디스패치.
- `render_txt.py`, `render_md.py`, `render_docx.py`.
- `templates/`: 회의록/보고서/기본 템플릿 정의.
- `naming.py`: 파일명 생성/중복 회피.

### 6.2 렌더링 규칙
- **txt**: 순수 텍스트. 여러 stage 포함 시 구분 헤더(옵션).
- **md**: 제목/섹션/목록 구조화. 3차 리포트는 heading·bullet 활용.
- **docx**: heading 스타일, 문단, 목록, 필요 시 표. 메타(날짜/모드/provider) 포함 옵션.

### 6.3 파일명/중복
- 기본 규칙 예: `{yyyymmdd-HHMM}_{모드명}_{stage}.{ext}`.
- 동일 파일 존재 시 `(1)`, `(2)` 증가 또는 덮어쓰기 확인(정책).

### 6.4 원자적 저장
- 임시 파일에 쓰고 완료 후 rename → 중간 실패 시 손상 파일 방지.

---

## 7. 오류/폴백/재시도 정책
- 경로 권한 없음/디스크 부족: 명확한 오류 반환.
- docx 렌더 실패: 원인 로그 + 사용자 알림, 다른 포맷 제안.
- 빈 산출물: 내보내기 거부 또는 경고.

---

## 8. 성능 목표
- 일반 문서 즉시 생성(수백 ms).
- 대용량 텍스트도 스트리밍/청크 없이 처리 가능한 수준.

---

## 9. 보안/프라이버시 고려사항
- 사용자가 지정한 위치에만 저장.
- 메타에 민감 정보(키 등) 미포함.

---

## 10. 테스트 항목
- 단위: 포맷별 렌더 결과 구조, 파일명 규칙, 중복 회피, 원자적 저장.
- 통합: 세션→3종 포맷 출력, docx 열림 검증.
- 엣지: 빈 stage, 특수문자/이모지 포함 텍스트, 매우 긴 리포트.

---

## 11. 관련 Phase와 구현 범위
- **P3**: txt/md/docx 전체 구현 + 기본 템플릿 + Workbench 연동.

---

## 12. 미결정 사항
- 회의록/보고서 docx 양식 세부 스타일.
- 중복 파일 기본 정책(증가 vs 확인).
- 여러 stage 한 파일에 합쳐 출력할지, stage별 분리할지 기본값.
