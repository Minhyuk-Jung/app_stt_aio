# C13. UI-Workbench 개발계획서

| 항목 | 내용 |
|------|------|
| 구성요소 ID | C13 |
| 이름 | UI-Workbench (작업대) |
| 계층 | UI (PySide6) |
| 관련 Phase | P2(부분), P3(핵심) |
| 상위 문서 | `README.md` |

---

## 1. 목적/책임
세션과 1/2/3차 산출물을 조회·편집·재가공·내보내기 하는 작업 공간. 기존 받아쓰기 앱과의 핵심 차별점.

- 세션 히스토리 목록/검색.
- stage별 산출물 조회·비교·편집.
- 재가공(reprocess) 실행.
- txt/md/docx 내보내기 실행.

**책임이 아닌 것**: 실제 가공(C4), 저장(C6), 파일 렌더(C8).

---

## 2. 선택 근거와 대안 검토
- **별도 작업대 필요**: 중간 산출물 관리/재가공 요구는 오버레이로 불가 → 독립 창.
- **편집 후 재가공 지원**: 1차 수정 → 2/3차 재실행 UX가 문서화 워크플로우의 핵심.

---

## 3. 공개 인터페이스 계약
- 발신: `request_reprocess(session_id, from_stage)`, `request_export(target, format, path)`, `update_artifact_text(artifact_id, text)`.
- 구독: `on_session_state`, `on_stage_completed`(실시간 갱신).

---

## 4. 입력/출력 데이터와 상태 전이
- 입력: C6 세션/artifact 조회, Application 이벤트.
- 출력: 사용자 편집/액션.
- 뷰 상태: 목록 → 상세(탭: 1차/2차/3차) → 편집/재가공/내보내기.

---

## 5. 의존 구성요소와 호출 관계
- **의존**: C6 Store(조회), C4 Pipeline(재가공, C10 경유), C8 Exporter(내보내기), C7 Mode(재가공 모드 선택).

---

## 6. 내부 설계
### 6.1 구조
- `workbench_window.py`: 메인 창/레이아웃.
- `session_list.py`: 히스토리 목록/검색/페이징.
- `artifact_view.py`: stage 탭, 비교/편집.
- `reprocess_panel.py`: from_stage/모드 선택 후 실행.
- `export_dialog.py`: 포맷/경로/템플릿 선택.

### 6.2 동작 원칙
- 조회/재가공은 워커에서 수행, 결과를 신호로 반영(UI 논블로킹).
- 편집 내용은 저장 시 C6 반영, 재가공 시 수정본을 입력으로 사용.
- 대량 히스토리는 페이징/지연 로드.

### 6.3 재가공 흐름
1. 세션 선택 → from_stage 지정(예: 1차 수정 후 2차부터).
2. 모드/프롬프트 확인 → 실행 요청.
3. 진행 표시 → 완료 시 해당 stage 갱신.

---

## 7. 오류/폴백/재시도 정책
- 재가공 실패: 기존 산출물 유지 + 오류 표시.
- 내보내기 실패: 사유 표시 + 다른 포맷 제안.
- 편집 미저장 이탈: 확인 프롬프트.

---

## 8. 성능 목표
- 목록/상세 로드 빠르게(인덱스/페이징).
- 대용량 텍스트 편집도 무리 없는 반응성.

---

## 9. 보안/프라이버시 고려사항
- 원문 텍스트를 표시하므로 화면 잠금/민감 세션 관리 고려(백로그).
- 내보내기 경로는 사용자 지정만.

---

## 10. 테스트 항목
- 단위: 목록/상세 렌더, 편집 반영, 액션 발신.
- 통합: 재가공→갱신, 내보내기→파일 생성.
- 엣지: 빈 히스토리, 대량 세션, 편집 충돌, 재가공 중 종료.

---

## 11. 관련 Phase와 구현 범위
- **P2**: 최소 조회(현재 세션 산출물 확인).
- **P3**: 히스토리/편집/재가공/내보내기 완성.

---

## 12. 미결정 사항
- stage 비교 뷰 방식(나란히 vs 탭).
- 편집 이력/버전 표시 여부(C6 설계 종속).
- 검색 필터 범위(날짜/모드/텍스트).
