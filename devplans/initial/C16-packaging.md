# C16. Packaging 개발계획서

| 항목 | 내용 |
|------|------|
| 구성요소 ID | C16 |
| 이름 | Packaging (빌드·배포) |
| 계층 | 통합/배포 |
| 관련 Phase | P3(핵심), P4/P5(확장) |
| 상위 문서 | `README.md` |

---

## 1. 목적/책임
Python 앱을 Windows 설치본/포터블 빌드로 패키징하고 CI로 자동화한다.

- PyInstaller/Nuitka로 실행 파일 생성.
- Inno Setup으로 설치 프로그램 작성.
- 모델 포함/제외 정책(기본 제외 + 최초 다운로드).
- CI 빌드 파이프라인.

**책임이 아닌 것**: 런타임 모델 다운로드(C18), 자동 업데이트(C22).

---

## 2. 선택 근거와 대안 검토
- **PyInstaller(안정) / Nuitka(성능·난독화)**: Python 앱의 현실적 배포 경로. P3에서 둘 중 안정성 우선 선택, 필요 시 Nuitka 전환.
- **Inno Setup**: 무료·성숙한 Windows 설치 제작기.
- **모델 미번들**: 용량/GPU 이슈로 설치본에 모델을 강제 포함하지 않음(벤치마크 표준).
- 대안 MSIX: 서명/스토어 정책 부담으로 후순위.

---

## 3. 공개 인터페이스(빌드 산출물 계약)
- 산출물: `STT-AIO-Setup-x.y.z.exe`(설치본), 선택적 포터블 zip.
- 빌드 스크립트: `build/` (spec, inno script, CI workflow).
- 버전 정보 주입(앱/설치본 동일 버전).

---

## 4. 입력/출력 데이터와 상태 전이
- 입력: 소스, 의존성 고정본(lock), 리소스(아이콘/PWA 정적파일/VAD 모델 등 소형 리소스).
- 출력: 설치본/포터블.
- 상태: 로컬 빌드 → CI 빌드 → 릴리스 아티팩트.

---

## 5. 의존 구성요소와 호출 관계
- **대상**: 전체 앱.
- **연계**: C18(모델은 런타임 다운로드), C22(업데이트가 설치본 형식 참조), C11(데이터 경로).

---

## 6. 내부 설계
### 6.1 구조
- `build/pyinstaller.spec` 또는 nuitka 설정.
- `build/installer.iss`(Inno Setup).
- `build/version.py`(버전 소스 단일화).
- `.github/workflows/build.yml`(CI, 예정).

### 6.2 패키징 절차
1. 의존성 고정 환경에서 빌드.
2. 실행 파일 + 필요한 소형 리소스 포함(대형 모델 제외).
3. Inno Setup으로 설치본 작성(시작메뉴/바로가기/제거 포함).
4. 버전/체크섬 기록.

### 6.3 정책
- 설치본은 경량 유지, 최초 실행 시 C18로 모델 준비.
- 오프라인 배포용 "모델 포함 번들"은 선택 산출물(대용량).
- 미서명 배포는 SmartScreen 경고 → 안내 문서(초기), 확대 시 코드사인(C22/P5).

---

## 7. 오류/폴백/재시도 정책
- 빌드 누락 의존성: CI에서 조기 실패.
- 실행 시 DLL/런타임 누락: 진단 로그(C20) + 재설치 안내.

---

## 8. 성능 목표
- 설치본 크기 최소화(모델 제외).
- 콜드 스타트 NFR(3초) 저해 요소 최소.

---

## 9. 보안/프라이버시 고려사항
- 신뢰 소스 의존성만 사용(lock/검증).
- 코드사인 미적용 기간 리스크 문서화.

---

## 10. 테스트 항목
- 설치/제거/재설치 스모크 테스트.
- 클린 Windows에서 실행(의존성 누락 확인).
- 업그레이드 설치(설정/DB 보존) 검증.

---

## 11. 관련 Phase와 구현 범위
- **P3**: 설치본 최초 작성 + 데이터 경로 확정 + 기본 CI.
- **P4**: PWA 정적 리소스 포함.
- **P5**: 코드사인/최적화.

---

## 12. 미결정 사항
- PyInstaller vs Nuitka 최종.
- 포터블 모드 제공 여부.
- 코드사인 도입 시점/인증서.
